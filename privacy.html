Privacy Policy

Last updated: December 6, 2024

This Privacy Policy describes how the GPT Backend Service ("we", "us", or "our") collects, uses, and shares information when you use our services through OpenAI's ChatGPT (Custom GPTs).

1. Information We Collect

We collect information solely to provide the technical functionality of the GPT actions (ZIP creation, Template rendering, Memory storage).

  • Input Data: Text, code, or content you explicitly send to the GPT for processing (e.g., to create a ZIP file or render a PDF).
  • Memory Data: Brand preferences (colors, fonts, brand names) that you ask the GPT to remember. This data is stored using a user-provided ID (e.g., Brand Code).
  • Log Data: Our hosting provider (Vercel) may collect standard server logs (IP address, browser type, timestamp) for security and debugging purposes.

2. How We Use Your Information

We use the collected information to:

  • Generate files (ZIPs, HTML, PDFs) as requested by you.
  • Store your design preferences to improve your experience in future sessions.
  • Maintain and improve the security of our backend services.

3. Data Storage and Sharing

We do not sell your personal data. We utilize the following third-party infrastructure providers:

  • Vercel Inc.: Hosting of the backend API (Serverless Functions).
  • Supabase (PostgreSQL): Database storage for the "Memory" feature.
  • OpenAI: The interface through which you interact with our services. Please refer to OpenAI's privacy policy for their data handling.

4. Data Retention

Temporary Files: ZIP bundles and rendered templates are generated on-the-fly or stored temporarily and are deleted regularly.

Long-term Memory: Data stored via the /memory endpoint remains stored until you overwrite it or request deletion.

5. Your Rights

You have the right to request information about your stored data or request its deletion. To do so, please contact the maintainer of this GitHub repository or simply instruct the GPT to overwrite/clear your memory data.

6. Contact

For privacy-related inquiries, please open an issue in the associated GitHub repository.
