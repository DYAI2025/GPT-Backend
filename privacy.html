Datenschutzrichtlinie

GPT Backend Services - Privacy Policy

1. Verantwortlicher

Diese Datenschutzrichtlinie gilt für den GPT Backend Service, bereitgestellt von DYAI2025.

Kontakt: privacy@dyai2025.com

2. Welche Daten werden verarbeitet?

Der Service verarbeitet folgende Daten:

  • API-Anfragen: Inhalte, die über die API gesendet werden (Texte, Dateinamen, Template-Daten)
  • Memory-Daten: Benutzer-ID und zugehörige Schlüssel-Wert-Paare (nur wenn der Memory-Service genutzt wird)
  • Generierte Dateien: ZIP-Archive und gerenderte Templates werden temporär gespeichert
  • Technische Daten: IP-Adressen, Zeitstempel, User-Agent (für Logging und Sicherheit)

3. Zweck der Datenverarbeitung

  • Bereitstellung der API-Dienste (ZIP-Erstellung, Template-Rendering, Memory-Speicherung)
  • Technische Wartung und Fehlerbehebung
  • Sicherheit und Missbrauchsprävention

4. Datenspeicherung

  • Memory-Daten: Werden in Supabase (PostgreSQL) gespeichert und bleiben erhalten, bis sie gelöscht werden
  • Generierte Dateien: Werden in Supabase Storage gespeichert und nach 30 Tagen automatisch gelöscht
  • Logs: Werden maximal 7 Tage aufbewahrt

5. Datenweitergabe

Daten werden nicht an Dritte verkauft. Folgende Dienste werden genutzt:

  • Supabase: Datenbank und Dateispeicherung (Server in EU/USA)
  • Vercel: Hosting der API (Server global verteilt)
  • OpenAI: Integration als GPT Custom Action (Anfragen werden von OpenAI weitergeleitet)

6. Ihre Rechte (DSGVO)

  • Auskunft: Sie können Auskunft über Ihre gespeicherten Daten verlangen
  • Löschung: Sie können die Löschung Ihrer Daten verlangen
  • Berichtigung: Sie können fehlerhafte Daten korrigieren lassen
  • Widerspruch: Sie können der Datenverarbeitung widersprechen

Kontaktieren Sie uns unter privacy@dyai2025.com

7. OpenAI GPT Integration

Dieser Service ist als Custom Action für OpenAI GPTs konzipiert. Bei Nutzung über einen GPT:

  • OpenAI leitet Ihre Anfragen an unsere API weiter
  • Wir speichern keine Konversationsverläufe
  • Die OpenAI Datenschutzrichtlinie gilt zusätzlich

8. Sicherheit

  • Alle Verbindungen sind HTTPS-verschlüsselt
  • API-Schlüssel werden serverseitig verwaltet
  • Regelmäßige Sicherheitsupdates

9. Änderungen

Diese Richtlinie kann aktualisiert werden. Die aktuelle Version ist immer unter dieser URL verfügbar.
